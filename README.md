# DonorSearch

**Модуль подбора комплектующих и анализа статистики использования аппаратных средств.**

Проект разрабатывается для автоматизации ремонта компьютерной техники, выявления неиспользуемого оборудования («доноров») и проверки совместимости комплектующих. Система может позволить сократить затраты на закупку нового оборудования за счет эффективного повторного использования имеющихся ресурсов.

## 🚀 Планируемые возможности

- **Интеллектуальный поиск доноров:** гибридный алгоритм поиска, учитывающий как физическую совместимость, так и фактическое время простоя оборудования.
- **Сетевой сканер:** многопоточный сбор данных об активности рабочих станций (через SSH/WinRM) для формирования статистики использования.
- **База знаний:** нормализованная база данных характеристик комплектующих (на основе данных PCPartPicker и Kaggle).
- **Проверка совместимости:** алгоритмы на базе SpEL (Spring Expression Language) для проверки совместимости процессоров, материнских плат, оперативной памяти и других компонентов.
- **Интеграция:** возможность встраивания в существующие системы инвентаризации предприятия.

## 🛠 Технологический стек

- **Backend:** Java 17+, Spring Boot.
- **Frontend:** React, TypeScript.
- **Database:** PostgreSQL.
- **Data Gathering:** Python (для парсинга PCPartPicker).
- **Tools:** Docker, Maven, Node.js, Vite.

## Процесс разработки (ветки)

Разработка ведется в изолированных ветках, что обеспечивает стабильность основной ветки `main`.

Активные ветки разработки:

- `knowledge-base` — реализация проектов, предназначенных для наполнения базы знаний о комплектующих.
- `frontend` — разработка клиентской части приложения.
- `backend` — разработка серверной части приложения.

_(Примечание: Названия веток могут незначительно отличаться, актуальный список см. во вкладке Branches)._
